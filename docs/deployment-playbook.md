# ClawBuilt Deployment Playbook

Complete step-by-step guide for deploying the ClawBuilt storefront to production on Vercel with all supporting services.

---

## 1. Prerequisites

- [ ] GitHub repo: `clawbuilt-ai` with Next.js 14 app
- [ ] Domain: `clawbuilt.ai` registered and DNS accessible
- [ ] Accounts created: Vercel, Stripe (live mode), Supabase, Cloudflare (R2), Resend, Cal.com

---

## 2. Supabase Setup

### 2.1 Create Project

1. Go to [supabase.com](https://supabase.com) → New Project
2. Name: `clawbuilt-prod`
3. Region: select closest to target audience (e.g., `us-east-1`)
4. Generate a strong database password — save it securely

### 2.2 Run Migration

1. Open SQL Editor in Supabase dashboard
2. Paste contents of `supabase/migrations/001_initial.sql`
3. Execute — creates `licenses`, `subscriptions`, `block_hours` tables with RLS

### 2.3 Collect Credentials

| Variable | Where to Find |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Settings → API → Project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | Settings → API → `service_role` key (secret) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Settings → API → `anon` key |

---

## 3. Stripe Setup

### 3.1 Activate Live Mode

1. Complete Stripe account activation (business details, bank account)
2. Toggle to **Live mode** in the dashboard

### 3.2 Create Products

Create all products per the pricing table in `clawbuilt-phase1-build-plan.md` (WS-02):

- 9 config products (3 verticals × 3 tiers: $497 / $997 / $1,497)
- 5 add-on products (Guided Setup through Harness Migration)
- 3 retainer products (Watchdog / Guardian / Command — recurring)
- 4 block hour products (5hr through 40hr)

Set metadata on each product: `vertical`, `tier`, `harness`, `type`, `service`, `plan`, `hours` as applicable.

### 3.3 Create Founding Client Coupon

1. Stripe Dashboard → Coupons → Create
2. Code: `FOUNDING50`
3. Discount: 50% off
4. Duration: once
5. Redemption limit: 10
6. Apply to: config products and add-on services

### 3.4 Webhook Configuration

1. Stripe Dashboard → Developers → Webhooks → Add endpoint
2. Endpoint URL: `https://clawbuilt.ai/api/webhook`
3. Events to listen for:
   - `checkout.session.completed`
   - `customer.subscription.created`
   - `customer.subscription.updated`
   - `customer.subscription.deleted`
4. Copy the webhook signing secret

### 3.5 Collect Credentials

| Variable | Where to Find |
|---|---|
| `STRIPE_SECRET_KEY` | Developers → API Keys → Secret key |
| `STRIPE_PUBLISHABLE_KEY` | Developers → API Keys → Publishable key |
| `STRIPE_WEBHOOK_SECRET` | Developers → Webhooks → Signing secret |

---

## 4. Cloudflare R2 Setup

### 4.1 Create Bucket

1. Cloudflare Dashboard → R2 → Create Bucket
2. Name: `clawbuilt-configs`
3. Location: Auto (or select US East)

### 4.2 Upload Config ZIPs

Upload config bundles to the following structure:

```
dental/dental-claw-starter.zip
dental/dental-claw-pro.zip
dental/dental-claw-agency.zip
style/style-claw-starter.zip
style/style-claw-pro.zip
style/style-claw-agency.zip
trades/trades-claw-starter.zip
trades/trades-claw-pro.zip
trades/trades-claw-agency.zip
```

### 4.3 Access Policy

- Bucket access: **Private** (no public access)
- All downloads served via presigned URLs generated by the webhook handler (1-hour expiry)

### 4.4 Create API Token

1. R2 → Manage R2 API Tokens → Create API Token
2. Permissions: Object Read & Write on `clawbuilt-configs`
3. Save the Access Key ID and Secret Access Key

### 4.5 Collect Credentials

| Variable | Where to Find |
|---|---|
| `R2_ACCOUNT_ID` | Cloudflare Dashboard → Account ID (right sidebar) |
| `R2_ACCESS_KEY_ID` | R2 API Token → Access Key ID |
| `R2_SECRET_ACCESS_KEY` | R2 API Token → Secret Access Key |
| `R2_BUCKET_NAME` | `clawbuilt-configs` |

---

## 5. Resend Setup

### 5.1 Domain Verification

1. Go to [resend.com](https://resend.com) → Domains → Add Domain
2. Domain: `clawbuilt.ai`
3. Add the DNS records Resend provides:
   - SPF (TXT record)
   - DKIM (CNAME records)
   - DMARC (TXT record, optional but recommended)
4. Verify — may take up to 48 hours for DNS propagation

### 5.2 Collect Credentials

| Variable | Where to Find |
|---|---|
| `RESEND_API_KEY` | API Keys → Create API Key |
| `RESEND_FROM_EMAIL` | `hello@clawbuilt.ai` (after domain verification) |

## 7. Vercel Deployment

### 7.1 Import Project

1. Go to [vercel.com](https://vercel.com) → Add New Project
2. Import from GitHub: `clawbuilt-ai`
3. Framework preset: Next.js
4. Root directory: `/` (default)
5. Build command: `next build`
6. Output directory: `.next`

### 7.2 Environment Variables

Add all environment variables in Vercel → Project Settings → Environment Variables.

**Required for production:**

```
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...

# Stripe
STRIPE_SECRET_KEY=sk_live_...
STRIPE_PUBLISHABLE_KEY=pk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Cloudflare R2
R2_ACCOUNT_ID=...
R2_ACCESS_KEY_ID=...
R2_SECRET_ACCESS_KEY=...
R2_BUCKET_NAME=clawbuilt-configs

# Resend
RESEND_API_KEY=re_...
RESEND_FROM_EMAIL=hello@clawbuilt.ai

# Cal.com
NEXT_PUBLIC_CAL_SCOPING_URL=https://cal.com/clawbuilt/scoping
NEXT_PUBLIC_CAL_GUIDED_URL=https://cal.com/clawbuilt/guided-setup

# App
NEXT_PUBLIC_SITE_URL=https://clawbuilt.ai
```

**Scope:** Set all variables to Production environment. Optionally add Preview/Development with test keys.

### 7.3 Deploy

1. Push to `main` branch — Vercel auto-deploys
2. Or trigger manually: Vercel Dashboard → Deployments → Redeploy

### 7.4 Custom Domain

1. Vercel → Project Settings → Domains → Add `clawbuilt.ai`
2. Vercel provides DNS records (A record or CNAME)
3. Update DNS at your registrar:
   - If using Vercel DNS: point nameservers to Vercel
   - If using external DNS: add the A record `76.76.21.21` or CNAME to `cname.vercel-dns.com`
4. Add `www.clawbuilt.ai` → redirect to `clawbuilt.ai`
5. SSL is auto-provisioned by Vercel

## 8. DNS Configuration

### 8.1 Required Records

| Type | Name | Value | Purpose |
|---|---|---|---|
| A | `@` | `76.76.21.21` | Vercel hosting |
| CNAME | `www` | `cname.vercel-dns.com` | www redirect |
| TXT | `@` | `v=spf1 include:...` | Resend SPF |
| CNAME | various | Resend-provided | Resend DKIM |
| TXT | `_dmarc` | `v=DMARC1; p=none; ...` | DMARC |
| MX | `@` | per email provider | Email forwarding |

### 8.2 Email Forwarding

Set up email forwarding at your registrar or use a service like ImprovMX:
- `hello@clawbuilt.ai` → personal email
- `support@clawbuilt.ai` → personal email

## 10. Rollback Procedure

### 10.1 Vercel Rollback

1. Vercel Dashboard → Deployments
2. Find the last known-good deployment
3. Click the three-dot menu → Promote to Production
4. This instantly routes traffic to the previous deployment

### 10.2 Database Rollback

If a migration causes issues:

1. Connect to Supabase SQL Editor
2. Identify and revert the problematic changes
3. Note: there is no automated rollback — always test migrations in a staging project first

### 10.3 Stripe Rollback

- Products and prices cannot be deleted once created (only archived)
- To "roll back" a price change: create a new price, update checkout links, archive the old price
- Webhook endpoint can be disabled in Stripe Dashboard → Webhooks

### 10.4 DNS Rollback

- DNS changes may take up to 48 hours to propagate globally
- Keep previous DNS records documented before making changes
- If pointing away from Vercel, update A/CNAME records at registrar
